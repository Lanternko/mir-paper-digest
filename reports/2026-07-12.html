Poly-SVC: Polyphony-Aware Singing Voice Conversion with Harmonic Modeling

騰訊 Lyra Lab 的零樣本跨語言唱聲轉換:不假設分離後人聲乾淨,用 CQT 整包保留主旋律與殘留和聲,配 MIDI 監督與 CFM 擴散解碼器,和聲場景 MOS 3.75 勝 SeedVC 的 3.35。

可轉譯 唱聲 pitch 人聲分離 flow/diffusion
Insight

洞見:真實歌曲經 UVR/Demucs 分離永遠留殘留和聲,與其硬擠出單一 F0,不如在表徵層改用 CQT 把「多條旋律線」整包交給生成器——CQT 的對數頻率軸還附送平移即轉調。這個思路對 karaoke-jp 的唱聲 pitch 抽取同樣成立:合唱與和聲段落的 F0 追蹤錯誤,可能該從表徵層解,而非事後修補;且論文顯示只需少量 MIDI 對齊資料做 L1 監督,就能教會 CQT encoder 壓掉音色等音高無關成分。

Try next

可做:在 karaoke-jp 的 pitch 模組加一條 CQT 路徑(12 bins/octave、共 84 bins、hop 441),對比 RMVPE 在和聲豐富的 J-POP 副歌片段的 pitch 曲線穩定度;若 CQT 更穩,再用手上的 MIDI 標註複製論文的 CQT-MIDI encoder L1 對齊監督。

重要度

Problem

SVC 系統依賴 F0 抽取器,但真實歌曲的人聲分離總留下殘留和聲,單一 F0 在多音高場景大量出錯,造成走音與頻譜失真;同時訓練用乾淨人聲、推論吃帶殘留和聲的分離結果,分佈不匹配。

Method

三個組件:(1) CQT-based pitch extractor,裁剪到 32Hz-1kHz 人聲域,保留主旋律與殘留和聲;(2) Random Sampler 用少量 MIDI 平行資料以 L1 損失讓 CQT encoder 貼近 MIDI encoder,抑制音高無關資訊;(3) 內容(Whisper-small)、音色(CampPlus)、音高三特徵經 length regulator 對齊後,條件到 U-DiT 的 Conditional Flow Matching 擴散解碼器,Firefly-GAN vocoder 出聲;另用 OpenVoice Timbre Shifter 防止內容特徵洩漏音色。

Data

語音用 Emilia 子集;唱聲用 m4singer、OpenSinger、OpenCpop、PopBuTFy、VocalSet(m4singer 帶 MIDI 標註);另以 UVR 從 70 小時中/英/粵/日混音歌抽人聲並去殘響,模擬帶殘留和聲的真實場景;評測 20 條 5-15 秒樣本。

Findings

12 人主觀評測:和聲場景 MOS 3.75、SIM-MOS 3.42,優於 SeedVC(3.35/3.40)、DDSP-SVC(2.98/2.82)、so-vits-svc(1.64/2.08);單旋律場景 MOS 3.98 亦全場最高。消融:移除 Timbre Shifter 使 SIM-MOS 明顯下降,移除 Random Sampler 則 MOS 與 SIM-MOS 同步變差;頻譜圖顯示能同時重建主旋律與和聲結構,SeedVC 只抓到主旋律且有明顯音高錯誤。

Limitations

評測全靠主觀 MOS(12 位中文受試者、20 條樣本),無任何客觀指標;和聲訓練資料是 UVR 分離的偽 ground truth;多人唱不同歌詞的內容重疊問題明言未解;訓練含日文但日文表現未單獨報告。

Editor Note

digest 之前談分離(Diff-VS、MSST)與唱聲轉譜(SongTrans、T3MS),這篇補上中間的縫:分離不完美時下游怎麼活。它的答案「殘留和聲不是雜訊、是該保留的資訊」對 karaoke-jp 的 pitch 抽取是可直接檢驗的假設,ICASSP 2026 已接收。

PairAlign: A Framework for Sequence Tokenization via Self-Alignment with Applications to Audio Tokenization

把 tokenization 重新定義為條件序列生成:AR decoder 自行決定 token 身分、順序、長度與 EOS,雙視角自對齊保留編輯距離結構;TIMIT 檢索 token 數省 55%。

貼近主線 codec/tokenizer sequence tokenization audio representation
Insight

洞見:現行 codec/VQ tokenizer 都是逐幀局部指派,序列層級的性質——緊湊度、長度控制、可終止、編輯相似度——從來不是訓練目標,只是副產品。PairAlign 把「同內容兩個視角應產生彼此高似然的序列」當成可規模化的編輯距離代理目標,順帶用競爭序列防 many-to-one 塌縮。對 MeanAudio 的意義:token 序列長度主宰生成模型算力,能學出「內容不變則序列不變」的可變長 tokenizer,比調 codebook 大小更根本;不過它目前只在 3 秒語音上驗證。

Try next

可做:給 MeanAudio 的評測管線加一個序列一致性檢查——對同一段音訊做 time-shift、增益等內容保持擾動,量現有 codec(如 DAC/EnCodec)token 序列的編輯距離分佈,先建立「現況有多不穩」的基線,再判斷自對齊式目標值不值得引入訓練。

重要度

Problem

比較、記憶、檢索、推理等操作天然定義在離散符號序列上;語言有現成 token,音訊的 token 得用學的。現有 audio tokenizer 靠量化、聚類或 codec 重建,token 逐幀指派,序列一致性、緊湊性、長度控制、終止與編輯相似度幾乎沒有被直接優化。

Method

把 tokenization 當條件序列生成:encoder 把音訊映成連續條件,AR decoder 從 BOS 生成 token 序列,學 token 身分、順序、長度與 EOS 位置。訓練用內容保持的雙視角:每個視角的序列要在對方的表徵下高似然,無關樣本提供競爭序列;從 VQ 式 tokenization 起步,再以 EMA-teacher 目標、跨視角 teacher forcing、prefix 汙損、似然對比與長度控制逐步精煉。

Data

3 秒語音片段訓練與分析;TIMIT 做編輯距離檢索評測;另設 continuous-sweep probe 檢查 100ms 位移下的局部重疊、長度控制與編輯軌跡;與稠密幾何式 tokenizer 對照。

Findings

學出緊湊、不退化、詞彙使用廣的序列,跨視角一致性強;TIMIT 檢索保留編輯距離搜尋能力,同時檔案庫 token 總數減少 55%;probe 顯示局部重疊低於稠密幾何 tokenizer,但長度控制更強、100ms 位移下編輯軌跡有界。

Limitations

只在 3 秒語音驗證,音樂與歌聲完全未測;評測以檢索與 probe 為主,沒接任何下游生成模型驗證 token 可用性;101 頁 preprint、under review,結果未經同儕審查;與主流 codec(EnCodec/DAC)的直接對比未見。

Editor Note

7/5 的可調 token rate codec 是在「幀率」維度鬆綁,這篇更激進:連 token 序列本身都改成生成式、可變長、以序列性質為目標。方向對 MeanAudio 的 codec 選型有啟發,但目前證據只到語音檢索,先當概念股觀察。
