Resonate: Reinforcing Text-to-Audio Generation via Online Feedback from Large Audio Language Models

把線上 GRPO 引入 flow matching 文字轉音訊模型,並改用 LALM(Qwen2.5-Omni)的 AQA yes/no 機率當細粒度 reward;470M 模型在 TTA-Bench 的音質與語義對齊都達 SOTA,勝過 MeanAudio 與 TangoFlux。

Insight

洞見:reward 模型比演算法更關鍵——CLAP 有 bag-of-words 效應、給的獎勵太粗;把獎勵改成 LALM 回答「音訊是否包含文字描述的聲音事件」的 yes 機率(AQAScore),一個 token 就能算、又貼近人耳判斷。且線上 GRPO 明顯勝過離線 DPO(AQAScore 0.737 vs 0.676),避免偏好資料與策略脫節的分布偏移。

重要度

Problem

TTA 的 RL 對齊多用離線 DPO(Tango2、TangoFlux 的 CRPO),偏好資料與訓練脫節導致分布偏移、探索受限;且 reward 幾乎都靠 CLAP,其 bag-of-words 效應讓時序與組合推理很弱,獎勵訊號粗糙、與人類偏好對不上。

Method

架構直接沿用 MeanAudio 的 Flux-style flow Transformer(16 MMDiT + 36 DiT,470M),先以 CFM 預訓練;再把 GRPO 適配到 flow matching 向量場上做線上 RL(Flow-GRPO,1000 步、G=24、噪聲 0.7、KL β=0.04);reward 採 AQAScore:問 LALM「音訊是否含 prompt 描述的聲音事件」,取 yes/no 的 softmax 機率。

Data

預訓練用約 370 萬對音訊-文字(1 萬小時:AudioCaps、AudioSet、Clotho、VGGSound、WavCaps、AudioStock);GRPO 只用 AudioCaps 訓練集的 prompt;評測用 TTA-Bench Accuracy 子集 1500 條多事件/時序複雜 prompt。reward 用 Qwen2.5-Omni、評測改用 Qwen3-Omni 以降低 reward hacking 風險。

Findings

TTA-Bench 上 AQAScore 0.737(MeanAudio 0.729、TangoFlux 0.677)、CLAP 0.476 皆最高,PQ 6.064 同步提升;消融顯示線上 GRPO 全面勝過 DPO 與 SFT(SFT 反而拉低音質),LALM reward 也優於 CLAP reward;人評 OVL/REL 均第一,而且只有 470M 參數、25 NFE。

Limitations

只在環境音/音效(AudioCaps 系)驗證,音樂與歌唱未測,AQA 式 reward 對音樂性(旋律、和聲)是否有效未知;reward 仍是代理模型,上限受 LALM 品質限制;GRPO rollout 需線上生成+打分,訓練成本比離線 DPO 高。

Editor Note

這篇幾乎是為 MeanAudio 的 reward alignment 路線量身打造:同架構、同預訓練目標,直接示範了「換掉 CLAP、上線上 RL」的完整配方,程式碼開源(xiquan-li/Resonate,含 pre-training/SFT/DPO/GRPO 全 pipeline)。優先讀消融兩張表。

STARS: A Unified Framework for Singing Transcription, Alignment, and Refined Style Annotation

首個統一框架一次完成歌唱標註四件事:音素級對齊、音符轉譜與定位、逐音素唱法技巧辨識、整體情緒/速度標註;取代 WhisperX+MFA+VOCANO 式的拼裝 pipeline,標註資料還能提升 SVS 自然度。

可轉譯 歌詞對齊 唱聲轉譜 風格標註
Insight

洞見:歌唱標註的錯誤主要來自「拼裝工具鏈的級聯誤差」——ASR、強制對齊、音高追蹤各管一段,邊界不一致會層層放大。STARS 用 frame/word/phoneme/note/sentence 五層階層式聲學特徵一次建模,讓對齊、音符、技巧、風格共享同一套邊界,非自迴歸局部聲學編碼器則保證各層表徵結構一致。

Try next

可做:抓 gwx314/STARS 開源模型,對 5-10 首人聲分離後的日文歌跑推論,重點量測音素邊界誤差與音符 onset/offset(對 mora-to-note timing 最關鍵),確認中英訓練的模型在日文上退化多少,評估是否值得用少量日文 TextGrid 微調。

重要度

Problem

SVS 與卡拉 OK 需要音素邊界、MIDI 音符、唱法技巧、整體風格四個層級的精細標註;人工標註極貴,現行自動標註各自為政(WhisperX/Qwen-Audio 轉錄、MFA 對齊、VOCANO/MusicYOLO 音高),級聯誤差大且標註粒度不一。

Method

階層式聲學特徵處理,依 frame→word→phoneme→note→sentence 五層逐級聚合;各層用非自迴歸局部聲學編碼器(U-Net + Conformer + FreqMOE)學結構化表徵;先做音素對齊與音符定位,再用逐音素特徵預測六類唱法技巧,最後聚合全部層級預測情緒、速度等全域屬性。

Data

用 GTSinger 的中文與英文子集訓練與評測(TextGrid 含詞/音素邊界、六種技巧逐音素標註、全域風格);評測指標含邊界誤差、音符 COn/COnOff、RPA、技巧 F1 與風格準確率。

Findings

在對齊、音符轉譜、技巧辨識、風格偵測各維度都優於 MFA、WhisperX、VOCANO 等既有工具組合;更重要的是用 STARS 自動標註的資料訓 SVS,人評自然度與風格控制精準度顯著提升,證明標註品質足以回饋生成。

Limitations

作者自承只在中文與英文驗證,日文等其他語言的泛化未測;依賴 GTSinger 規模的乾淨獨唱資料,對真實混音、合唱或人聲分離殘留的魯棒性未知;六類技巧分類未必涵蓋 JPOP 常見唱法。

Editor Note

karaoke-jp 的資料標註瓶頸(音素對齊+音符 timing 一次搞定)正是這篇的目標場景,ACL 2025 Findings、程式碼與 demo 都開源。日文未驗證是最大風險,但 kana 音素集比英文單純,遷移值得一試;可與上月的 VocalParse 做同題對比(NAR 階層式 vs LALM 自迴歸)。
