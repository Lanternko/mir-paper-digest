Neural Audio Codec with Adjustable Token Temporal Resolution Using Sampling-Frequency-Independent Convolutional Layers

用取樣頻率無關卷積讓單一 neural codec 支援多種 token 時間解析度(TTR),把 token rate 與音質細節的取捨變成可調參數,不必為每種解析度分開訓練 codec。

貼近主線 codec/tokenizer text-to-audio/music token rate
Insight

洞見:把 TTR 視為 token 序列的取樣週期,從共享參數動態生成各解析度的卷積 kernel,一個模型覆蓋多種 token rate。這直接對應 MeanAudio 在 codec 端「序列長度 vs 快速聲學事件」的老問題。

重要度

Problem

Neural codec 的 token 時間解析度(TTR)控制「表現快速聲學事件」與「縮短 token 序列」的取捨,但多數 codec 只能在單一 TTR 訓練,每換一種解析度就要重訓一次。

Method

提出取樣頻率無關(SFI)卷積機制:把 TTR 當作 token 序列的取樣週期,從共享參數集生成 TTR 相依的卷積 kernel,並隨 TTR 調整 kernel size 與 stride;直接嵌進 Descript Audio Codec(DAC),quantizer 不動。

Data

在環境音重建上實驗;基於 DAC 架構。注意:論文用的是環境音,不是音樂——音樂與歌聲上的表現需要自行驗證。

Findings

單一模型在多種 TTR 下的重建品質,優於「每種 TTR 換一組專屬層」的 single-model baseline;代表共享參數生成 kernel 的做法確實可行。

Limitations

只在環境音重建驗證,未測音樂、歌聲或下游生成任務;quantizer 未動,TTR 改變對 RVQ codebook 使用率與語義保留的影響未知;也還沒接到 token LM 上驗證生成品質。

Editor Note

與 MeanAudio 的 codec/token rate 問題直接相關,也呼應 DAC 語義保真度研究線。下一步:拿它的 SFI 機制對 DAC 做單變因替換,先在音樂資料上跑重建+語義檢索 smoke test。

Decomposer: Learning to Decompile Symbolic Music to Programs

把 MIDI 反編譯成可執行、可編輯的 Strudel 音樂程式:先用合成配對資料做 SFT,再用 RL 同時獎勵重建忠實度與程式可讀性,避免退化成逐音符轉寫。

MIR 觀點 MIDI/score symbolic music RL reward
Insight

洞見:低資源配對資料的解法是「合成資料 SFT + 無配對資料 RL」兩段式;reward 同時管忠實度與可讀性,防止模型走捷徑。這個 recipe 可遷移到歌詞對齊或 mora-to-note 這類配對資料稀缺的任務。

重要度

Problem

從演奏(symbolic MIDI)反推高階音樂指令是逆問題:Strudel 是低資源語言,幾乎沒有 MIDI-程式配對資料;只優化 MIDI 重建又會塌縮成不可讀的逐音符轉寫。

Method

兩階段後訓練:(1)建 Strudel-Synth 合成語料(配對的 Strudel 程式與渲染 MIDI)做監督微調;(2)在無配對 MIDI 上做強化學習,reward 同時計 MIDI 重建忠實度與程式碼可讀性。

Data

Strudel-Synth 合成語料+合成與真實世界 MIDI benchmark;專案頁有展示(yewon-kim.com/decomposer)。

Findings

MIDI 重建忠實度大幅超過閉源 LLM 直接生成,程式可讀性與多樣性優於啟發式轉換器;證明小模型後訓練在這種結構化逆問題上能贏過通用大模型。

Limitations

任務限定在 symbolic(MIDI)層,不含音訊轉錄;Strudel 表達力有限,對複雜表演性演奏(rubato、微時值)的處理未知;可讀性 reward 的設計細節需要看正文驗證。

Editor Note

CMU Donahue/Neubig 組。對主線的價值在訓練 recipe 而非任務本身:合成配對+RL 雙 reward 的路子,值得記進 karaoke-jp 的歌詞對齊實驗選項。
