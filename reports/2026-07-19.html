Local Multimodal Music Alignment from Global Supervision

重要度

Problem

理解音樂需要跨模態的局部對應（例如演奏音訊時間點對應樂譜影像位置），但密集局部對齊標註極難取得；實務上往往只有較粗的全域配對（整段音訊與整頁／整段樂譜）。

Method

提出 FuSiLi（Fused Sinkhorn-Localized Similarity）：在局部 image patch 與 audio frame 特徵上以 Sinkhorn 軟對齊計算相似度，用於多模態對比學習。再以 hybrid 目標（FuSiLi + 傳統全域相似度）微調預訓練 CLIP 與 CLAP 編碼器，對 raw sheet music 影像與音訊配對。

Data

以樂譜影像–音訊配對微調 CLIP/CLAP；在跨模態檢索與 frame-level alignment 任務上對比多組全域／局部 baseline。ISMIR 2026。

Findings

FuSiLi 能在僅有全域監督下學到有效局部關係；局部對齊任務優於多組 baseline，同時在檢索上仍與傳統全域對比學習競爭。顯示「局部可學、全域不崩」可並存。

Limitations

主實驗是 sheet music 影像而非日文歌詞文字／kana；遷移到 LRC 或 mora-to-note 需另驗。Sinkhorn 對齊成本與超參敏感度摘要未詳；真實 JPOP 混音與版權樂譜場景未覆蓋。

Editor Note

karaoke-jp 今日首選：對齊主線從「要金標時間戳」退到「只要段落配對」。可與 CTCLRC／yohane 串成「弱監督粗對齊 → forced align 精修」。MeanAudio 弱連結：CLAP 微調與跨模態局部相似度可反哺 caption–audio 對齊診斷。

WanSong v1.0 Technical Report

WanSong 以純 diffusion（非 AR、非 AR+diffusion 串級）一次生成最長約 5 分鐘多語歌曲，並同時輸出人聲與伴奏雙 stem；支援 step-distillation 加速與微調客製化編輯路徑。

貼近主線 song generation diffusion dual stems
Insight

洞見:MeanAudio 長期在「codec token AR」與「latent diffusion」之間擺盪；WanSong 把「長篇歌曲 + 雙 stem 一次出」壓進單一 diffusion 框架，直接打中 karaoke 後端的「要乾淨伴奏＋可選人聲」需求。相對 Qwen-Music（7/15）的 tokenizer–LLM–render 三件套，這是另一條 pure diffusion 路線——工程上可比較：可控性、長序列穩定度、與 reward／DPO 後訓練的介面。

重要度

Problem

音樂生成 foundation model 要同時兼顧高效生成、高保真長時音訊與可控性仍困難；AR 或 AR+diffusion 串級管線在長篇歌曲與編輯性上各有瓶頸。

Method

提出 WanSong：純 diffusion 一次直接生成高保真、多語、最長約 5 分鐘的歌曲，並在單次前向輸出雙 stem（vocals + background music）。擴散框架支援 step-distillation 加速推論，並提供微調／客製化路徑以支援下游編輯。

Data

技術報告（Wan Team）；摘要未列公開訓練集規模與基準分數細節。arXiv:2607.14749，2026-07-16。

Findings

報告定位為 commercial-grade 長篇歌曲生成：相對 AR 與多階段串級，純 diffusion 可一次出長篇高保真與雙 stem，並以 distillation 與 fine-tune 路徑兼顧速度與客製。

Limitations

技術報告摘要缺少客觀指標表、與 SOTA 的定量對照與開源承諾；「commercial-grade」需聽感與授權側另驗。雙 stem 是否達可用 karaoke 級分離未在 abs 證明。

Editor Note

MeanAudio 今日首選：長篇 + 雙 stem 是 karaoke 伴奏生成的最短路徑之一。可與 7/15 Qwen-Music、7/17 FdAudio 並列成「歌曲生成三路線」：LLM-token、pure diffusion、FD 後訓練。karaoke-jp 強連結在 BGM stem。
