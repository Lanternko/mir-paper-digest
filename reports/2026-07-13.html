SegTune: Structured and Fine-Grained Control for Song Generation

快手 Kling 團隊的 DiT+流匹配歌曲生成:全局 prompt 管風格、段落 prompt 管局部情緒與配器,再用微調 Qwen3-4B 直接生成 LRC 句級時間戳;PER 14.5% 全場最低,ACL 2026 口頭報告並獲最佳論文提名。

貼近主線 text-to-music flow/diffusion 歌詞對齊
Insight

洞見:歌曲生成的可控性瓶頸其實在時間軸——SegTune 把 LRC 時間戳當成 LLM 的生成目標(微調 Qwen3-4B,句級 MAE 0.99 秒;GPT-4o zero-shot 是 3.24 秒),證明句級 timing 可以從歌詞+風格描述被直接預測出來。對 karaoke-jp 來說,LRC 之上多了一條「先驗生成」路線,可當 forced alignment 失敗時的 fallback 或 QA 異常偵測。另一個硬結論:prompt encoder 用 Qwen3-Embedding 性別控制 92.2%、MuQ-MuLan 只有 47.6%——MuLan 系 embedding 對人聲屬性近乎全盲,因為訓練 caption 根本沒標歌手屬性。

Try next

可做:用 karaoke-jp 手上的日文 LRC 資料微調小型 LLM(照論文配方:Qwen3-4B-Base + LoRA rank 32、8 epochs)做「歌詞+曲風→句級時間戳」預測,對比 forced alignment 輸出量 MAE;若 <1.5 秒,即可上線當對齊結果的 sanity check。

重要度

Problem

現有歌曲生成(AR 的 YuE/LeVo、NAR 的 DiffRhythm+/ACE-Step)只吃全局 prompt,無法控制段落級的情緒、配器、節奏演變;NAR 模型還依賴人工標註或 zero-shot LLM 的歌詞時間戳,錯誤多且阻礙互動使用。

Method

DiT(1.1B、16 層 LLaMA-style blocks)+ 條件流匹配,44kHz 音訊由 1D VAE 壓到 21.5Hz latent。三路條件:全局 prompt 廣播全曲、段落 prompt 只廣播到對應時間窗(皆用 Qwen3-Embedding-0.6B 編碼後 channel 拼接)、歌詞經 G2P 轉音素後按時間戳寫入 placeholder 序列。時間窗與歌詞 timing 來自微調 Qwen3-4B 的 duration predictor,直接輸出 LRC 格式句級時間戳。訓練三階段:預訓練→SFT→兩輪迭代 DPO。

Data

內部策展語料,>90% 為華語流行歌;資料 pipeline:Audiobox-Aesthetics/SongEval 過濾品質、Demucs v4 分離人聲、FireRedASR(中)/Whisper-large-v3(其他語言)轉錄、all-in-one 模型抽段落結構、Audio Flamingo 3 生成全局與段落 caption;duration predictor 用超過 10 萬條 LRC 歌詞微調。

Findings

PER 14.5% 全場最低(YuE 48.5%、ACE-Step 35.6%、DiffRhythm+ 27.4%);性別控制 96.7% 最高;DPO 後主觀音樂性 MOS 4.57±0.52 全場第一(Wilcoxon p<.001)。消融:段落 prompt 拼接明顯優於 global-only 與加權混合;duration predictor MAE 0.99 秒遠優於 GPT-4o 的 3.24 秒,且下游音樂性對 timing 誤差相當穩健。

Limitations

訓練與評測幾乎全在華語流行歌,測試集僅 15 首 ChatGPT 生成歌詞,語言泛化未驗證;段落結構模糊時 duration predictor 退化並拖累全曲;DPO 後性別/年齡控制下降(偏好資料偏向年輕女聲);段內漸強、裝飾音等 intra-segment 動態仍不可控;訓練語料未開源。

Editor Note

digest 追過 DiffRhythm 系與 ACE-Step,SegTune 是這條 NAR 歌曲生成線上第一個把「段落級文字控制 + LLM 時間戳預測」做成完整系統的;主觀評測僅 5 位聽者、樣本偏小,但客觀指標與消融夠紮實。程式碼在 KlingAIResearch/SegTune。

EntangleCodec: A Unified Discrete Audio Tokenizer via Semantic-Acoustic Entanglement

復旦的統一離散 audio tokenizer:量化前就把語義與聲學縫進同一表徵,用 LLM 生成的 rich caption 做對比對齊取代 ASR 轉錄;單 codebook 50 TPS,理解任務 MMAR 比最佳 codec 基線高 7.4%,重建 UTMOS 3.96。

貼近主線 codec/tokenizer caption/data evaluation
Insight

洞見:語義流與聲學流分開再融合不是必然——把對齊監督從 ASR 轉錄升級成涵蓋歌手、情緒、韻律、場景的 rich caption,單一 token 流就能同時餵理解與生成。消融最有說服力:拿掉對比損失,UTMOS 從 3.96 崩到 3.04,語義對齊反而救了重建品質。另一個訊號:0.6B LM 配好 tokenizer 打贏 13B+ 連續表徵模型,表徵品質比堆參數划算——對資源有限的 MeanAudio 是好消息。

重要度

Problem

重建導向 codec(EnCodec/DAC)缺語義、下游推理弱;語義導向 tokenizer(SpeechTokenizer/XCodec 系)靠雙流融合,帶來冗餘與錯位;且語義監督多半只有 ASR 轉錄,丟失歌手、情緒、韻律、聲學場景等非文字訊息。

Method

單一 Transformer encoder 從 Mel 圖直接學統一表徵,[CLS] 全局向量與 LLM 自動生成的 rich caption 做 CLIP 式對比學習;經 MLP 投影 + L2 正規化後餵單 codebook VQ(50 tokens/s);解碼用 Rectified Flow 的 Llama-style diffusion decoder 重建 Mel。兩階段訓練:先聯合訓 encoder+quantizer+decoder,再凍結前端只精修 decoder。下游凍結 tokenizer,接 Qwen3 系列 LM 做 audio QA/TTS/TTA。

Data

訓練:LibriSpeech、MusicBench、AudioSet、AudioCaps、WavCaps;理解評測 MMAR、MMAU、MMAU-mini;TTS 用 SEED-TTS 測試集(Whisper-large-v3 量 WER),TTA 用 CLAP score;codec 對照全部統一 Qwen3-0.6B backbone 隔離 tokenizer 效果。

Findings

理解:同 backbone 對照平均 34.5 全場最高,MMAR 34.3 比次佳高 7.4 點;生成:TTS WER 9.8%、UTMOS 3.89 均為 codec 基線第一;重建:UTMOS 3.96 僅次 XCodec2 的 4.02,是唯一在語音/音效/音樂三域都名列前茅的 tokenizer;scaling:0.6B 贏 13B+ 連續表徵 LLM,8B 在 MMAR 42.6% 創新高。

Limitations

語義粒度受 caption 品質上限,細微情緒變化、複雜和聲未必捕捉得到;音樂重建仍輸無語義對齊的 XCodec;text-to-music 未直接評測(TTA 以一般音效為主);scaling 只驗到 8B,重建與理解對容量的需求分配未探索。

Editor Note

digest 已追過 LoSATok、PairAlign 與 Token-Space Manipulation 這條 tokenizer 線,EntangleCodec 的差異點是把 rich caption 對比對齊放進量化前表徵,且程式碼與權重已開源,適合直接當 MeanAudio codec 選型的對照組。
